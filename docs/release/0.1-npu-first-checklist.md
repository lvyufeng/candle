# Candle 0.1 NPU-First Release Checklist

## Scope

This checklist validates Candle 0.1 NPU-first release readiness:

- Ascend 910B single-card golden training loop,
- NPU checkpoint continuity,
- CPU + contract CI fallback safety.

## Required Commands

### CPU + Contract Baseline

```bash
pytest tests/cpu/ tests/contract/ -v --tb=short
```

Expected:
- No failures in CPU/contract suites.

### NPU Golden Loop Gate

```bash
pytest tests/npu/test_npu_golden_training_loop.py -v --tb=short
```

Expected:
- PASS (or SKIP on non-NPU runner),
- Loss trace generated by test logic.

### NPU Checkpoint Continuity Gate

```bash
pytest tests/npu/test_npu_training_checkpoint_continuity.py -v --tb=short
```

Expected:
- PASS (or SKIP on non-NPU runner),
- Reloaded state can continue finite training.

### NPU Scalar Mul Regression Gate

```bash
pytest tests/npu/test_mul_scalar_regression_npu.py -v --tb=short
```

Expected:
- PASS (or SKIP on non-NPU runner),
- `randn -> mul(scalar)` path remains stable.

## Optional Packaging Smoke

```bash
pip install -e .
python -m pytest tests/cpu/test_import.py -q
```

Expected:
- Editable install succeeds,
- Import smoke passes.

## Go/No-Go Criteria

Go only if all are true:

- CPU + contract baseline green,
- NPU golden loop gate green on 910B runner,
- NPU checkpoint continuity gate green on 910B runner,
- NPU scalar-mul regression gate green on 910B runner,
- No unresolved P0 issues for declared 0.1 scope.

Hard rule:

- No NPU runtime fallback to CPU is allowed in operator execution paths.

No-Go if any of the above fails.
