# Candle 0.1 NPU-First Release Checklist

## Scope

This checklist validates Candle 0.1 NPU-first release readiness for the declared GA surface only:

- Ascend 910B single-card golden training loop,
- NPU checkpoint continuity,
- NPU no-CPU-fallback release paths,
- CPU + contract CI baseline.

Experimental distributed/HCCL validation stays visible, but it is not a 0.1 merge blocker yet.

## Required Local Repro Commands

### CPU + Contract Baseline

```bash
pytest tests/cpu/ tests/contract/ -v --tb=short
```

Expected:
- No failures in CPU/contract suites.

### NPU 0.1 Release Gate Set

```bash
PYTHONPATH=/usr/local/Ascend/ascend-toolkit/latest/python/site-packages:src \
pytest \
  tests/npu/test_npu_golden_training_loop.py \
  tests/npu/test_npu_training_checkpoint_continuity.py \
  tests/npu/test_mul_scalar_regression_npu.py \
  tests/npu/test_no_cpu_fallback_npu.py \
  -v --tb=short
```

Expected:
- PASS on Ascend 910B runner,
- no CPU fallback in declared NPU release paths,
- checkpoint save/load path stays finite and deterministic enough for regression gating.

### Experimental Distributed Validation

```bash
PYTHONPATH=/usr/local/Ascend/ascend-toolkit/latest/python/site-packages:src \
pytest tests/distributed/test_hccl_mvp_longrun.py -v --tb=short
```

Expected:
- Informational only for 0.1,
- PASS on stable multi-card Ascend runner if the job is triggered.

## Required CI Jobs

The following CI jobs are required before merge for changes that touch release-critical code paths:

- `pylint-check`
- `test-cpu`
- `test-mps`
- `test-npu`

The following CI job is non-blocking and experimental:

- `test-distributed-experimental`

## Go/No-Go Interpretation

Go only if all required CI jobs above are green and all required local repro commands match their expected outcomes.

No-Go if any of the following is true:

- `test-npu` fails,
- `test-cpu` fails,
- `pylint-check` fails,
- a declared NPU GA path still performs runtime CPU fallback,
- a known P0 NPU issue is undocumented,
- the distributed experimental job is the only failing job but the change claims distributed GA support.

Hard release rule:

- No NPU runtime fallback to CPU is allowed in operator execution paths.
